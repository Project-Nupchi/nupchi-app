# Project Nupchi Design Guide

이 문서는 Project Nupchi의 현재 제품 UI를 기준으로 한 지속형 디자인 컨텍스트다. 새 화면을 설계하거나 기존 화면을 수정할 때 홈과 수조 목록의 시각 언어를 출발점으로 삼고, 임의의 값보다 코드에 정의된 디자인 토큰과 공용 컴포넌트를 우선한다.

- 마지막 확인: 2026-07-10
- 제품: 제주 광어 양식장용 수조 상태 기록·AI 분석 앱
- 주 플랫폼: 세로형 iOS 앱(Expo Router/React Native), 웹 대응
- Figma 기준: [Project Nupchi 디자인, `46:450`](https://www.figma.com/design/mby1o9uMiY4tYstDx81vp5?node-id=46-450)
- 홈 구현 기준: [`src/app/index.tsx`](src/app/index.tsx)
- 수조 목록 구현 기준: [`src/app/tank-status.tsx`](src/app/tank-status.tsx)
- 토큰 기준: [`src/constants/aqua-theme.ts`](src/constants/aqua-theme.ts)

## Design Context

### Users

주 사용자는 제주 광어 양식장의 관리자와 현장 작업자다. 사용자는 양식장 안에서 수조 상태를 빠르게 훑고, 이상이 의심되는 개체를 촬영하고, 수조·수조군 단위의 점검 이력을 추적한다.

핵심 작업 흐름은 다음과 같다.

1. 어가 컨텍스트로 로그인한다.
2. 홈에서 오늘의 최고 위험과 우선 확인할 수조를 파악한다.
3. 수조를 선택하고 광어를 촬영하거나 사진을 가져온다.
4. AI 분석의 대기·완료·실패 상태와 개체별 병변 후보를 확인한다.
5. 재촬영, 수조군 동선 분리, 기록 보존, 전문가 확인 등 다음 행동을 결정한다.

이 제품은 진단 도구가 아니라 현장 관찰과 추적을 돕는 참고 도구다. AI 결과는 확진으로 표현하지 않으며, 격리·투약·출하 제한·신고는 관리자와 전문가의 최종 판단 영역으로 남긴다.

### Brand Personality

브랜드 성격은 **차분함, 신뢰감, 현장성**이다.

- 차분함: 물빛 그라디언트와 넓은 여백으로 일상 점검의 안정감을 만든다.
- 신뢰감: 상태, 근거, 촬영 시각, 수조군 맥락을 숨기지 않고 명확히 보여준다.
- 현장성: 짧은 카피, 큰 터치 영역, 위험 우선 정렬, 즉시 촬영 CTA로 다음 행동을 빠르게 만든다.

위험은 과장된 공포나 화면 전체의 빨간색으로 표현하지 않는다. 평소의 차분한 수면 이미지는 유지하고, 경고색·병변 표시·경고 넙치·물결 효과를 필요한 위치에만 집중한다.

### Aesthetic Direction

현재의 미감은 **light-first aqua glass**다.

- 화면 전체는 상단의 선명한 파랑에서 하단의 흰색으로 내려오는 물빛 그라디언트를 사용한다.
- 상단 그라디언트에는 흰색 텍스트, 하단의 밝은 영역과 카드 안에는 짙은 청남색 잉크 텍스트를 사용한다.
- 표면은 흰색 70–90%의 반투명 카드, 흰색 헤어라인, 20px 모서리, 부드러운 청남색 그림자로 구성한다.
- Pretendard의 굵고 짧은 제목과 친근한 한국어 존댓말을 사용한다.
- 모래색 넙치 일러스트와 단순한 상태 아이콘을 사용해 수산 도메인을 친근하게 드러낸다.
- 경고 표현은 빨간 상태 배지, 경고 넙치, 붉은 동심원, 병변 점선 박스처럼 의미가 있는 곳에만 쓴다.
- 카메라 화면의 어두운 크롬은 촬영 집중을 위한 문맥적 예외이며, 별도의 전역 다크 모드를 의미하지 않는다.

일반적인 SaaS 관리 대시보드, 네온 중심의 AI 제품, 병원용 임상 UI, 게임처럼 장식적인 수산 캐릭터 화면으로 드리프트하지 않는다. 정보 밀도보다 우선순위와 다음 행동이 먼저 보이는 모바일 현장 도구여야 한다.

### Design Principles

1. **최고 위험과 다음 행동을 먼저 보여준다.** 경고 → 의심 → 양호 순으로 정렬하고, 가장 중요한 수조로 바로 이동할 수 있게 한다.
2. **차분한 화면 안에서 위험만 선명하게 만든다.** 상태색은 국소적으로 사용하며, 나머지 화면은 물빛과 흰 글래스 표면으로 안정감을 유지한다.
3. **토큰과 의미를 재사용한다.** 색상·간격·라운드·그림자·타이포를 화면마다 다시 만들지 않는다.
4. **AI의 확률성과 근거를 숨기지 않는다.** “후보”, “의심”, “참고치”를 사용하고 원본 사진, 병변 위치, 촬영 시각, 수조군 맥락과 함께 보여준다.
5. **현장 조작을 빠르고 포용적으로 만든다.** safe area, 충분한 터치 영역, 짧은 라벨, 색상 외 아이콘·텍스트 단서를 함께 사용한다.

## Source of Truth

디자인 결정을 구현할 때 다음 우선순위를 따른다.

1. 현재 작업의 명시적 제품 요구사항
2. Figma의 화면 구조와 시각 의도
3. `src/constants/aqua-theme.ts`의 `FigmaTokens`
4. 같은 파일의 시맨틱 별칭 `Palette`, `Gradient`, `Radius`, `Space`, `Shadow`, `Type`, `Water`
5. 홈·수조 목록의 검증된 화면 패턴
6. 제품용 공용 컴포넌트
7. `src/constants/copy.ts`의 사용자 노출 문구

구현에서는 `FigmaTokens`의 원시 값을 화면에 직접 복사하지 말고 시맨틱 별칭을 import한다. 필요한 의미가 없다면 해당 화면에서 하드코딩하지 말고 `aqua-theme.ts`에 별칭을 추가한다. Figma에서 읽힌 개별 효과와 코드 토큰이 다를 때도 카드에는 `Shadow.card`, 내비게이션과 FAB에는 `Shadow.navigation`을 사용한다.

### 제품 UI의 기준이 아닌 항목

- `src/constants/theme.ts`, `ThemedText`, `ThemedView`, `HintRow`, `WebBadge`, `ExternalLink`, `Collapsible`, `AnimatedIcon*`는 Expo 템플릿 계열이며 현재 제품 화면의 Aqua 디자인 기준이 아니다.
- `src/global.css`의 Spline Sans/Inter 스택은 웹 스타터 설정이다. 제품 타이포의 의도는 `Type`의 Pretendard다.
- `assets/images/icon.png`, `splash-icon.png`, `favicon.png`의 “A” 마크와 Expo 로고는 Nupchi의 확정 브랜드 에셋으로 간주하지 않는다.
- `app.json`은 `userInterfaceStyle: automatic`이지만 제품용 어두운 카드·배경 토큰은 없다. 새 다크 모드를 임의로 추론하지 않는다.

## Product and Status Model

홈의 시각 우선순위는 실제 도메인 로직과 연결돼 있다.

- 수조의 기본 상태는 가장 최근 점검 결과다.
- 같은 `groupId`의 다른 수조가 `suspicious`이면, 자체 결과가 양호한 수조도 계통 위험 때문에 `caution`으로 표시한다.
- 정렬은 `suspicious` → `caution` → `normal`, 같은 상태에서는 최근 촬영순이다.
- 홈 헤드라인은 경고가 하나라도 있으면 경고 수만 보여주고, 경고가 없을 때 의심 수를 보여준다. 둘 다 없을 때만 전체 양호 메시지를 쓴다.

내부 enum과 사용자 레이블을 혼동하지 않는다.

| 내부 값 | 사용자 레이블 | 의미 | 텍스트 | 배경 | 아이콘 |
| --- | --- | --- | --- | --- | --- |
| `normal` | 양호 | 정상·좋음 | `#0CB26A` | `#2CB77A26` | 초록 체크 |
| `caution` | 의심 | 주의·연동 위험 | `#F78009` | `#F7800926` | 주황 삼각형 |
| `suspicious` | 경고 | 이상·우선 확인 | `#EE2C1D` | `#F7534626` | 빨강 번개 |

건강 상태와 처리 상태는 다른 개념이다.

- 건강 상태: `StatusBadge`로 양호·의심·경고를 표현한다.
- 처리 상태: `StateChip`으로 판정 대기(파랑)·판정 완료(초록)·판정 실패(빨강)를 표현한다.
- 개체 단위 이진 판정에서는 `normal`과 `suspicious`만 사용하며, `StatusBadge binary`가 비정상을 “의심”으로 통합한다.

## Visual System

### Color and Surface

화면 배경 그라디언트는 다음 고정 스톱을 사용한다.

| 위치 | 색상 |
| ---: | --- |
| 0% | `#4B9AE9` |
| 20% | `#78B3EF` |
| 40% | `#A5CCF4` |
| 62% | `#D2E6FA` |
| 88% | `#FFFFFF` |

주요 시맨틱 색상은 다음과 같다.

| 역할 | 토큰 | 값 |
| --- | --- | --- |
| 그라디언트 위 기본 텍스트 | `Palette.onGradient` | `#FFFFFF` |
| 그라디언트 위 보조 텍스트 | `Palette.onGradientMuted` | `#FFFFFFCC` |
| 카드 기본 텍스트 | `Palette.text` | `#1E2A45` |
| 카드 보조 텍스트 | `Palette.textMuted` | `#1E2A45B2` |
| 카드 약한 텍스트 | `Palette.textSubtle` | `#1E2A4580` |
| 기본 글래스 | `Palette.glass` | `#FFFFFFB2` |
| 강조 글래스 | `Palette.glassStrong` | `#FFFFFFE5` |
| 글래스 테두리 | `Palette.glassLine` | `#FFFFFFCC` |
| 기본 액션 | `Palette.primary` | `#3689DD` |
| 눌린 액션 | `Palette.primaryPressed` | `#1D75CD` |
| 부드러운 액션 배경 | `Palette.primarySoft` | `#E9F2FC` |
| 어두운 FAB·잉크 | `Palette.ink` | `#1E2A45` |

글래스 효과는 플랫폼별로 다르게 구현한다.

- 웹: `backdrop-filter`를 사용하며 공용 카드는 20px, 홈 내비게이션·FAB는 15px blur를 기준으로 한다.
- 네이티브: 실제 배경 blur에 의존하지 않고 반투명 흰색 표면과 테두리로 같은 인상을 만든다.
- regular와 strong 글래스의 차이로 계층을 만들고, 카드마다 새로운 불투명도를 만들지 않는다.

### Typography

의도된 제품 서체는 Pretendard다. 모든 새 텍스트는 가능한 한 `Type` 토큰을 spread해서 사용한다.

| 토큰 | 크기/행간 | 굵기 | 자간 | 대표 용도 |
| --- | --- | ---: | ---: | --- |
| `Type.display` | 28/39 | 700 | -0.84 | 홈 상태 헤드라인 |
| `Type.appBarTitle` | 24/34 | 600 | -0.48 | 상단 앱바 제목 |
| `Type.title` | 20/28 | 600 | -0.40 | 위치·주요 제목 |
| `Type.heading1` | 20/26 | 700 | -0.20 | 카드 수조 ID |
| `Type.heading2` | 18/25 | 600 | -0.36 | 섹션 제목·보조 헤드라인 |
| `Type.body1` | 16/22 | 600 | -0.32 | 주요 본문 |
| `Type.body2` | 14/20 | 500 | -0.14 | 메타데이터 |
| `Type.button` | 16/22 | 700 | -0.32 | 버튼 라벨 |
| `Type.fieldLabel` | 14/20 | 600 | 0 | 입력 라벨 |
| `Type.label1` | 14/20 | 500 | -0.28 | 상태·작은 라벨 |
| `Type.label2` | 13/18 | 600 | -0.26 | 배지 |
| `Type.label3` | 11/15 | 500 | -0.22 | 하단 탭 |
| `Type.caption` | 13/18 | 400 | 0 | 촬영 시각·설명 |

저장소에는 현재 Pretendard 폰트 파일과 명시적 로딩 코드가 확인되지 않는다. `Type`은 디자인 의도의 기준이지만, 실제 기기에서 정확한 재현이 필요한 작업에서는 폰트 번들·로딩 여부를 별도로 검증한다.

### Spacing, Radius, and Shadow

- 간격 스케일: `2, 4, 8, 16, 20, 28, 36`
- 화면 좌우 여백: 기본 20px
- 주요 세로 리듬: 20px
- 카드 내부 여백: 보통 16–20px
- 카드 사이 간격: 8–16px, 목록 기준 12px
- 카드 모서리: 20px
- 이미지 모서리: 12px
- 입력·일반 버튼 모서리: 16px
- 둥근 CTA: 40px
- 배지·탭·FAB: `Radius.pill`
- 일반 카드: `Shadow.card`
- 떠 있는 내비게이션·FAB·고정 dock: `Shadow.navigation`

### Imagery and Icons

제품 고유 시각 자산의 기준은 `assets/images/home`이다.

- `flounder.png`: 모래색의 단순하고 친근한 넙치. 기본·양호·의심 맥락에 사용한다.
- `flounder-warn.png`: 몸 뒤쪽으로 붉어지는 경고 넙치. 최고 상태가 경고일 때만 사용한다.
- `tank-thumbnail.png`: Figma 수조 목록 카드의 56×56 수조 썸네일. 목록에서는 홈 히어로 넙치 대신 이 이미지를 사용한다.
- `plus.png`: 수조 목록 앱바의 24×24 파란 추가 아이콘. 40px 글래스 원 안에 배치한다.
- `ripple`: 경고를 암시하는 낮은 불투명도의 붉은 동심원. 홈은 100/81/60%의 세 원으로 재현한다.
- 상태 아이콘: 체크·삼각형·번개를 상태색 및 텍스트와 함께 사용한다.
- 내비게이션·카메라·chevron: 단순한 24–28px 실루엣과 라운드 캡을 유지한다.
- 홈의 Figma SVG는 CSS 변수와 blur 호환 문제 때문에 네이티브에서 3x PNG를 사용한다.
- PNG 상태 아이콘은 원색을 유지하고, 단색 탭·FAB 아이콘만 `tintColor`를 사용한다.
- 실제 분석 사진에서는 병변 후보를 붉은 반투명 점선 박스와 짧은 라벨로 표시한다.

### Motion and Feedback

모션은 상태 이해를 방해하지 않는 짧은 피드백에 한정한다.

- 일반 press: opacity 0.7–0.85
- 버튼 press: 필요할 때만 scale 0.985 정도
- FAB press: scale 0.95
- 선택 상태는 색·표면·레이블 변화로 즉시 확인할 수 있게 한다.
- 장식적 반복 애니메이션을 기본값으로 추가하지 않는다.
- 새 모션은 reduced-motion 경로를 함께 고려한다.

## Component Language

### Canonical Components

| 컴포넌트 | 역할과 시각 계약 |
| --- | --- |
| `ScreenShell` | 물빛 그라디언트 위 세로 ScrollView. 좌우 20px, 요소 간 20px. FAB나 dock이 있으면 `bottomInset`으로 겹침을 피한다. |
| `GlassCard` | regular=`Palette.glass`, strong=`Palette.glassStrong`, 흰 1px 선, 20px radius, `Shadow.card`. 웹만 20px blur. |
| `ActionButton` | primary/secondary/danger/ghost, regular/compact. 기본 높이 56, compact 42, radius 16. 의미는 항상 텍스트 라벨에 둔다. |
| `StatusBadge` | 건강 상태용 pill. 상태색 배경·선·점·텍스트를 함께 사용하고 compact/binary 변형을 지원한다. |
| `StateChip` | 분석 처리 상태용 pill. pending/completed/failed를 blue/success/danger로 분리한다. |
| `Section` | 바깥 제목, `GlassCard` 본문, 선택적 각주를 하나의 리듬으로 묶는다. 카드 안은 20px padding, 16px gap. |
| `PhotoAnalysisStage` | 4:3 분석 이미지, 20px radius, cover 이미지 또는 넙치 placeholder, 완료된 실제 사진에만 병변 박스 overlay. |
| `FlounderMark` | View 조합의 비례형 넙치 실루엣. 상세·placeholder용이며 선택적으로 병변을 표시한다. 장식 요소로 취급한다. |

새 제품 컴포넌트는 위 컴포넌트를 조합하거나 같은 Aqua 토큰 계층을 따른다. 오래된 컴포넌트에 남아 있는 하드코딩 색상·폰트 값을 새 코드에 복제하지 않는다.

### Recurring Screen Patterns

- **홈 미니 수조 카드:** 가로 스크롤, 최소 폭 94px, 수조 ID + 상태 아이콘 + 상태명만 표시한다.
- **수조 목록 카드:** 최소 높이 104px, 56×56 이미지, 수조 ID와 배지, 한 줄 메타데이터와 최근 촬영 시각을 표시한다.
- **선택 카드:** strong glass에 primary 2px outline과 `accessibilityState.selected`를 사용한다.
- **기록 카드:** 상단에 날짜와 상태, 본문에 두 줄 요약, 필요할 때 위험색 질병 태그를 둔다.
- **폼 필드:** 높이 56px, radius 16, 흰 솔리드 표면과 약한 잉크 테두리, 명시적 라벨을 사용한다. 수조 입식 정보용 여러 줄 입력은 104px를 기준으로 한다.
- **고정 dock:** 콘텐츠 최대 폭과 정렬하고 safe area 및 스크롤 하단 여백을 함께 확보한다.
- **하단 내비게이션:** 64px 높이의 글래스 pill, 80×56 탭, 24px 아이콘, 11px 라벨. 활성 탭만 흰 capsule을 가진다.
- **카메라 FAB:** 64×64 어두운 원, 흰 1px 선, 28px 카메라 아이콘, `Shadow.navigation`을 사용한다.

홈과 수조 목록의 하단 내비게이션은 동일한 시각 계약을 공유한다. 공용 컴포넌트로 추출할 수 있지만, 추출 과정에서 위치·크기·활성 capsule·safe-area 동작을 바꾸지 않는다.

## Home Screen Reference

홈은 앱의 디자인과 정보 우선순위를 가장 압축적으로 보여주는 기준 화면이다. 공용 `ScreenShell`과 `GlassCard`를 직접 사용하지는 않지만, Figma 일치를 위해 같은 `Gradient`, `Palette`, `Radius`, `Space`, `Shadow`, `Type`을 사용한다.

### Information Hierarchy

위에서 아래의 순서를 유지한다.

1. **어가 컨텍스트:** 지도 핀과 `session.farmName`으로 현재 양식장을 확인한다.
2. **오늘의 상태:** 작은 kicker와 두 줄짜리 큰 헤드라인으로 최고 위험을 요약한다.
3. **즉시 행동:** “수조 바로 확인하기”로 위험 정렬의 첫 수조 상세에 이동한다.
4. **상태 히어로:** 넙치와 동심원으로 상태를 감성적으로 시각화한다.
5. **수조 목록 미리보기:** 위험순 가로 카드로 각 수조의 핵심 상태를 훑는다.
6. **고정 주 내비게이션:** 홈·수조 목록 탭과 독립된 촬영 FAB를 제공한다.

### Layout Contract

- 세로 스크롤 안의 본문은 중앙 정렬하고 최대 폭을 520px로 제한한다.
- 본문 좌우 여백은 20px다.
- 수조 카드는 full-bleed 가로 스크롤이지만 첫 카드의 시작점은 중앙 본문과 정렬한다.
- 넙치는 뷰포트 폭의 68%를 사용하되 최대 300px다.
- 동심원은 넙치보다 크게 두되 콘텐츠를 밀어내지 않는 absolute 장식으로 둔다.
- CTA는 48px 높이의 strong glass pill이다.
- 하단 탭과 FAB는 safe area 위 약 20px에 고정한다.
- 스크롤 하단에는 고정 내비게이션과 겹치지 않도록 약 140px를 확보한다.
- 상단·하단 고정 요소의 값은 `useSafeAreaInsets`로 조정한다.

### State Variants

- 경고 수조가 있으면 경고 수를 헤드라인에 보여주고 `flounder-warn.png`를 사용한다.
- 경고 없이 의심 수조만 있으면 의심 수를 헤드라인에 보여주되 기본 넙치 이미지를 유지한다.
- 경고와 의심이 모두 없으면 “모든 수조가 양호한 상태예요”를 사용한다.
- CTA는 가장 위험한 수조 상세로 이동하며 대상 수조가 없으면 disabled다.
- 미니 카드는 위험도와 최근성 순서로 배치하고 각각 해당 수조 상세로 이동한다.

### Home Copy Tone

- 헤드라인은 “상태예요”, “발견됐어요”처럼 짧고 친근한 존댓말을 사용한다.
- 주요 버튼은 “확인하기”, “촬영하기”, “저장”처럼 명확한 행동형을 사용한다.
- 내비게이션은 “홈”, “수조 목록”, “촬영”처럼 짧은 명사형을 사용한다.
- 큰 헤드라인의 줄바꿈은 카피 문자열에서 의도적으로 관리한다.
- 위험 상황에서도 공포를 조장하지 않고 바로 실행할 수 있는 다음 행동을 제시한다.
- 질병과 AI 결과는 “후보”, “의심”, “유사”, “확률적 참고치”처럼 비확정 언어를 사용한다.
- 사용자 노출 문구는 JSX에 새로 하드코딩하지 않고 `src/constants/copy.ts`에서 관리한다.

### Do Not Generalize Current Gaps

다음은 현재 홈 구현의 동작이지만 새 화면의 디자인 원칙으로 복제하지 않는다.

- “수조 목록” 제목과 chevron은 이동할 것처럼 보이지만 현재 Pressable이 아니다.
- 홈은 비활성 수조도 집계·노출하지만 전체 수조 목록은 활성 수조만 보여준다.
- 수조가 0개여도 현재는 전체 양호 메시지를 사용하며 별도 empty state가 없다.
- 홈에는 로딩·오류·샘플 데이터 모드가 직접 드러나지 않는다.

이 항목을 건드리는 작업에서는 제품 의도를 확인한 뒤 상호작용과 상태를 명시적으로 설계한다.

## Related Screen Guidance

- **수조 목록:** Figma `46:450`의 직접 기준이다. 393×852 세로 프레임, 20px 좌우 여백, 12px 카드 간격, 상단 24px 제목, 40px 추가 버튼, 하단 탭과 FAB를 사용한다.
- **수조 상세:** strong glass 헤더에 수조 ID, 수조군, 입식 정보, 상태와 주요 CTA를 모으고 아래에 시간순 점검 기록을 쌓는다.
- **카메라:** 촬영 전 수조 선택은 Aqua glass를 유지하고, 실제 촬영 뷰에서만 어두운 크롬과 흰 가이드를 사용한다.
- **분석 결과:** 사진·개체 필터·근거·대응을 단계적으로 보여준다. 대기·실패·완료를 각각 독립된 상태 화면으로 처리한다.
- **AI 대응 제안:** 번호가 있는 현장 단계, 공식 출처 링크, 확진이 아니라는 고지를 명확히 구분한다.
- **수조 추가/편집:** 시스템 헤더를 숨긴 전체 화면 폼이다. 옅게 wash된 물빛 배경, safe-area 아래 60px 커스텀 앱바, 최대 520px 폼, 고정된 취소/저장 2버튼 footer를 사용한다.

## Responsive and Accessibility Rules

- 앱은 portrait 우선이지만 고정 픽셀 화면으로 만들지 않는다.
- 일반 콘텐츠는 최대 520px, 카메라·결과의 넓은 컨테이너는 최대 560px를 기준으로 중앙 정렬한다.
- 스크롤 콘텐츠에는 고정 탭·FAB·dock의 실제 높이만큼 하단 여백을 둔다.
- 아이콘 단독 버튼에는 명확한 `accessibilityLabel`과 `accessibilityRole`을 제공한다.
- 탭·필터·선택 카드에는 `accessibilityState.selected`를 제공한다.
- 상태는 색상만으로 전달하지 않고 아이콘 또는 점, 텍스트 레이블을 항상 함께 사용한다.
- 장식용 넙치·동심원과 버튼 내부의 장식 아이콘은 접근성 트리와 pointer interaction에서 제외하고, 의미는 바깥 컨트롤의 라벨로 제공한다.
- 주요 조작은 44px 이상의 실효 터치 영역을 확보한다. 40px 컨트롤은 `hitSlop` 또는 외부 터치 영역을 보완한다.
- 고정 높이 카드와 한 줄 말줄임은 큰 글자 설정에서 잘리지 않는지 검증한다.
- 웹에서는 hover뿐 아니라 `focus-visible`을 제공한다.
- 비동기 분석의 로딩·완료·실패 전환은 스크린 리더가 인지할 수 있는 알림 방식을 고려한다.
- 공식 WCAG 목표는 저장소에서 확인되지 않았으므로, 새 요구사항이 없다면 적어도 위 기준을 회귀 방지선으로 삼는다.

## Implementation Checklist

새 화면이나 컴포넌트를 만들기 전에 확인한다.

- [ ] 홈 또는 수조 목록의 정보 계층과 실제로 연결되는가?
- [ ] 색·간격·radius·shadow·type을 `aqua-theme.ts`에서 가져왔는가?
- [ ] 새 의미가 필요하면 화면 하드코딩 대신 시맨틱 토큰을 추가했는가?
- [ ] 건강 상태와 처리 상태를 구분했는가?
- [ ] 경고를 색상 하나에만 의존하지 않았는가?
- [ ] `GlassCard`, `ScreenShell`, `ActionButton`, `StatusBadge`, `StateChip`, `Section`을 재사용할 수 있는지 확인했는가?
- [ ] safe area와 고정 내비게이션에 필요한 스크롤 여백을 확보했는가?
- [ ] 모바일 최대 폭과 넓은 화면의 중앙 정렬을 검증했는가?
- [ ] 빈 상태, 로딩, 오류, disabled, pressed, selected 상태가 있는가?
- [ ] 사용자 문구를 `src/constants/copy.ts`에 두고 비확정 AI 언어를 유지했는가?
- [ ] 아이콘 단독 조작의 접근성 라벨과 44px 터치 영역을 확인했는가?
- [ ] 실제 기기에서 Pretendard 로딩과 큰 글자 레이아웃을 확인했는가?

## Open Assumptions

다음 내용은 코드와 현재 화면에서 추론했으며 별도 브랜드 문서로 확정되지 않았다.

- 브랜드의 세 단어를 “차분함·신뢰감·현장성”으로 정의했다.
- 현재 제품은 단일 light-first 테마로 간주했다.
- 공식 접근성 목표가 없어 현재 구현과 모바일 기본 원칙을 기준으로 삼았다.
- Pretendard는 의도된 서체지만 번들·로딩 상태는 별도 검증 대상으로 남겼다.

이 가정이 제품 방향과 다르면 이 문서의 `Design Context`와 관련 규칙을 먼저 갱신한 뒤 화면을 수정한다.
